3. Horizontal vs Vertical Scaling
Vertical Scaling (Early Phase)

Upgrade machine:

2 CPU → 8 CPU
8GB → 32GB RAM

Cheap and simple.

Horizontal Scaling (Real Scaling)

Duplicate services:

API Server 1
API Server 2
API Server 3

Traffic distributed automatically.

Your future system MUST support horizontal scaling.

4. Stateless Backend Design

Golden rule:

Servers must remember NOTHING.

All state stored externally.

Bad:

User session stored in server memory

Crash = logout.

Correct:

Session → Database / Redis

Any server can serve any user.

Enables infinite scaling.

5. Database Scaling Strategy

Your biggest future bottleneck = DATA.

Phase 1 — SQLite

Perfect for MVP.

Phase 2 — PostgreSQL Migration

Needed when:

concurrent users increase

analytics queries grow

Phase 3 — Read Replicas

Split workload:

Primary DB → Writes
Replica DB → Dashboard Reads

Analytics never blocks submissions.

Phase 4 — Data Sharding

Massive scale:

Org A → DB shard 1
Org B → DB shard 2

Prevents database overload.
